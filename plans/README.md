### Deviations from the plans as executed

The plans were the starting point; the actual implementation went further to meet
the project goal (always compile the newest canasta, add real prices + categories):

- **R/ module layer**: instead of a single `R/clean_canasta_data.R`, the shared
  logic was split into `R/clean_canasta_data.R`, `R/categories.R`, `R/ipc.R`,
  composed by `R/compile_canasta.R`. All three scripts source it.
- **Validation gate (001)** validates the full enriched schema (categories, IPC,
  real prices). The monthly-contiguity check was downgraded from a hard failure
  to a NOTE — INIDE legitimately skips months (e.g. Mar 2026 is missing), and a
  hard fail would permanently block the pipeline.
- **Features added beyond the plans**: `categoria` (3 official groups),
  `ipc`/`ipc_estimado`/`precio_real`/`total_real` (IMF IFS monthly CPI via
  DBnomics, constant 2024 córdobas), a nominal-vs-real toggle and category tab in
  the app, two new figures, and a "Cómo usar estos datos" README section.
- **Data updated** to Apr 2026 (3 new months ingested live: Ene/Feb/Abr 2026).

## Findings considered and rejected

(So nobody re-audits these.)

- **Legacy URL spaghetti in `01_files.R` / `02_scrape.R`**: historical
  one-shot scripts; the data they fetched is already captured in
  `data/monthly/`. Rewriting buys nothing.
- **`.Rhistory` / `.DS_Store` / `.Rproj.user` / `rsconnect/` in the
  worktree**: all already gitignored and untracked — no action.
- **`anti_join` on (year, month) never re-downloads revised INIDE files**:
  by design — avoids monthly churn; revisit only if INIDE is observed
  revising published months.
- **renv-based deploy rebuild**: real waste (~10–15 min/deploy, unpinned),
  but changing it needs a live deploy to verify — deferred, see plan 005's
  maintenance notes rather than rejected outright.
- **CI R version pin (4.3.2)**: works; bump deliberately, not as part of an
  automated plan (see plan 005 maintenance notes).
- **`FIXES.md` / `CHANGES_SUMMARY.md` cleanup**: harmless historical
  records of the Nov 2025 incident; deleting them loses context.
- **Suspected Nov/Dec 2025 data duplication**: investigated — Dec 2025 is
  genuine new data (29 of 53 prices differ from Nov; adjacent months
  sharing some prices is normal for this series).

## Direction findings (not planned — maintainer's call)

Surfaced during the audit; each would be a design/spike plan if selected:

1. **Real (inflation-adjusted) prices** — every plot is explicitly
   "Precio nominal" over an 18-year span; deflating with BCN/INIDE CPI would
   answer the question most users of this series actually have. Effort: M
   (coarse), needs a CPI source decision.
2. **Category dimension** — the app sidebar describes the basket's three
   official categories (alimentos, uso doméstico, vestimenta) but the data
   has no category column; the official 53-item structure maps statically by
   the `row` id. Enables category-level cost views. Effort: S–M.
3. **Programmatic data access docs** — README advertises RDS/CSV/Stata
   formats but never gives the stable raw URLs or a citation hint. A short
   "Cómo usar estos datos" section (raw GitHub URLs, R/Stata one-liners,
   suggested citation) is cheap and serves the actual audience. Effort: S.
